在开发中我们会经常切换一些分支，而切换时候你本地会有一些更改了的代码，但是这些代码还没有写完整，所以我们一般会进行 `git stash save` 本地保存一下。
然后等后面代码切换回来之后，再 `git stash pop`。
这天正常 `git stash save` 之后，脑袋一抽 命令输成了 `git stash clear`。当时心中暗道： “md! 坏了”。
平复下后，感觉应该是能找回的，毕竟有 `git reflog` 这种东西，遂在网上寻找解决方案。

首先，如果你执行的是 `git stash drop` 命令，终端会输出这条 stash 的 hash。
你直接 `git stash apply {hash}` 就可以了。如下图所示：

![img](/static/images/git/git-stash-apply-drop-hash.png)

那么直接可以完结撒花了。

如果你执行的是 `git stash clear`，那就需要使用 `git fsck` 命令慢慢找回了。
`git fsck` 加上 `--unreachable` 参数，可以显示出所有不可访问的对象。

接下来我执行 `git fsck --unreachable`，可以看到如下图所示：

![img](/static/images/git/git-fsck--unxx.png)

这么多 commit hash ， 关键它还是无序的，我并不知道那个是误删除的，只能一个一个去 `git show {hash}`。

我这里下载了 vscode 插件 Gitlens Inspect。可以在终端 command + 鼠标左键 查看。
一个一个查看，终于找到了它

![img](/static/images/git/git-fsck-target-commit-hash.png)

使用下面这条命令可以更直观的看到。

```sh
git log --graph --decorate --pretty=oneline --abbrev-commit --all $(git fsck --no-reflogs | grep commit | cut -d' ' -f3)
```

![img](/static/images/git/git-fsck-log-target-hash.png)

找到了之后，直接 `git stash apply {hash}` 就可以了，终于可以真正的 <GradientText>完结撒花✿✿ヽ(°▽°)ノ✿</GradientText>了。

什么？你想问我采用了什么方法？因为我误删除之后什么操作都没做，直接 command + z 撤销回来了。【狗头】
